# Contributing to KitWeight

Thanks for helping improve KitWeight.

## Before opening a change

- Search existing issues before filing a duplicate.
- Keep pull requests focused on one problem or feature.
- For substantial behavior or design changes, open an issue first so the
  direction can be agreed before implementation.
- Never include real pack lists or other personal data in fixtures or screenshots.

## Development workflow

1. Fork and clone the repository.
2. Install dependencies with `npm ci`.
3. Create a branch from `main`.
4. Make the change and update documentation when behavior changes.
5. Run `npm run check` before opening a pull request.

Pull requests should explain the user-visible effect and how the change was
tested. Screenshots are useful for layout changes at both desktop and mobile
widths.

By contributing, you agree that your contributions will be licensed under the
project's MIT License.
